STEP 0

깨끗한 출발점 만들기

"처음부터 다시"라면, 정말로 빈 폴더에서 시작해야 합니다. 단, 옛 코드는 버리지 않고 참고용으로 보존합니다.

목표

  • 기존 프로젝트 위에 덧칠하지 않고 빈 루트에서 새로 시작한다.
  • 그러면서도 main 브랜치와 옛 코드는 안전하게 남겨둔다.
  • 다시 쓸 가치가 있는 자산(도서관 URL 맵, 파싱 규칙)은 손 닿는 곳에 둔다.

왜 새 브랜치인가, 왜 orphan인가

같은 저장소 안에서 새 시도를 하려면 새 브랜치를 파는 게 자연스럽습니다. 그래야 main의 안정된 상태를 건드리지 않고 자유롭게 실험할 수 있으니까요.

왜 일반 브랜치가 아니라 orphan?

일반 git checkout -b는 main의 커밋 이력을 그대로 물려받습니다. --orphan은 이력이 0인 브랜치를 만듭니다. 우리가 만들 첫 커밋이 이 브랜치의 진짜 첫 줄이 되죠. "from scratch"라는 의도와 가장 잘 맞습니다. (main의 역사는 main에 그대로 남아 있으니 잃는 건 없습니다.)

해본 것

1. orphan 브랜치 생성

git checkout --orphan v2-solar
git reset           # 옛 파일들을 스테이지에서 내려놓음 (작업트리는 그대로)

이 시점에 디스크의 파일은 그대로지만, git 입장에선 "아직 아무것도 커밋되지 않은 새 브랜치"입니다.

2. 옛 코드를 docs/v1/ 로 이동

mkdir -p docs/v1
git mv 00_src README.md app.py requirements.txt docs/v1/
mv "BookToss_시연영상2.mp4" *.hwp *.pdf docs/v1/   # 추적 안 되던 자료들도 함께
왜 지우지 않고 옮기나

재구축의 두 핵심 자산 — 서울 도서관 100여 곳의 포털 URL을 담은 catalog_index.yaml, 그리고 구청 사이트별 HTML 파싱 규칙 — 은 처음부터 다시 만들 필요가 없습니다. 바퀴를 재발명하지 않고 "참고해서 골라 옮기는" 것이 현명한 재시작입니다.

3. 큰 바이너리는 git에서 제외

시연 영상(18MB)·PDF·HWP는 디스크엔 두되 저장소엔 넣지 않습니다. .gitignore로 비대화를 막습니다.

# .gitignore
.env
__pycache__/
docs/v1/*.mp4
docs/v1/*.hwp
docs/v1/*.pdf
docs/v1/00_src/data/raw/

4. 출발점 커밋

git add -A
git commit -m "chore: start v2-solar from scratch; archive v1 under docs/v1"

결과

v2-solar (orphan, 커밋 1개)
├── README.md          ← v2 의도 + 진행표
├── .gitignore         ← .env, 캐시, 큰 미디어 제외
└── docs/
    └── v1/            ← 기존 해커톤 프로젝트 통째로 보존
        ├── app.py, 00_src/, requirements.txt
        └── 00_src/configs/catalog_index.yaml  ← 나중에 재활용

루트는 docs/와 .git만 남아 사실상 비었습니다. 진짜 from scratch.

짚고 갈 점

git log v2-solar를 찍어보면 커밋이 딱 1개입니다. v1의 first 커밋 이력을 물려받지 않았다는 증거죠. 같은 저장소 안에 두 개의 독립된 역사가 공존하게 됩니다 — main(원본)과 v2-solar(재구축).
